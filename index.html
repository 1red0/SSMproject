Limbajul de programare Python


    Python este un limbaj de programare dinamic multi-paradigmă, creat în 1989 de programatorul olandez Guido van Rossum. Van Rossum este și în ziua de astăzi un lider al comunității de dezvoltatori de software care lucrează la perfecționarea limbajul Python și implementarea de bază a acestuia, CPython, scrisă în C.

Capitol I



Funcția print



O functie poate avea:
1. un efect (de exemplu: reda o melodie, deseneaza un crocodil, afiseaza un text)
2. un rezultat (de exemplu: returneaza resultatul ridicarii la putere o valoare, returneaza lungimea unui text)

Observatie: Conventie de scriere
Pentru a distinge cuvintele obișnuite de numele funcțiilor, vom pune o pereche de paranteze goale după numele lor, chiar dacă funcția corespunzătoare necesită unul sau mai multe argumente (de exemplu, vom scrie print()).


Functiile pot fi:
  • incorporate (build-in) (de exemplu, functia print() sau min() sau max())
    •     print(max(1,2,3))
        print(min(1,2,3))
  • din module cu care vine Python sau module instalate separate
  • proprii - scrise de tine (numele date funcției ar trebui să fie semnificativ, sa insemne ceva)


    • O functie poate avea 0 sau mai multe argumente.

    Pentru functia print() singurul argument este un String (sir de caractere) - tipul de date String este delimitat cu ghilimele:
        print("Buna!")

    Numele functiei, impreuna cu parenteze si agrumentele sale reprezinta invoarea functiei. La invocarea unei functii, Python:
  • verifică dacă numele functiei este legal (adica este definita/exista undeva; daca nu exista, se arunca eroare)
  • verifică dacă se trimit argumentele necesare (tipul si numarul de argumente; daca nu se respecta signatura, se arunca eroare)
  • sare in codul functiei invocata cu argumentele date si ele transmite functiei
  • executa codul functiei (provoaca efectul dorit sau evalueaza rezultatele) si isi incheie sarcina
  • revine în locul imediat după invocare și își reia execuția


    • Efectul urmatorului fragement de cod:
        print()
    este ca adauga la consola o linie noua. Practic este echivalent cu:
        print("\n")
    Functia print() poate fi invocata si mai multe argumente:
        print("nimeni nu scrie","poezii","cu ochii deschisi")
    Rezultatul va fi:
        nimeni nu scrie poezii cu ochii deschisi

    Rezulta ca functia print() cu mai multe argumente:
  • le afiseaza pe o singura linie
  • adauga un spatiu intre argumente (ca asa vrea el)

    • Mecanismul cheie de argument (keyword arguments) permite schimbarea comportamentului unei functii. Forma pentru o cheie de argument (cheie de argument) este:
  • cuvantul cheie care identifica argumentul (ex: end)
  • =
  • valoare (ex: " si ")


    • Cheie de argument: end

    De exemplu, pentru functia print() putem specifica sa nu mai puna \n (line noua) dupa afisarea la consola a argumentelor. In exemplul de mai jos am inlocuit "\n" cu " si ":
        print("Afara este", "noapte", end=" si ")
        print("pustiu.")
     Rezultatul va fi:
        Afara este noapte si pustiu.

    Cheie de argument: sep

    Alta cheie de argument pentru functia print() este sep (separator)
        print("nu", "mai", "pune", "spatii", "intre noi", sep="-")
     Rezultatul va fi:
        nu-mai-pune-spatii-intre noi

    Deci, in Python, argumentul /parametrul unei functii poate fi:
  • pozitional (positional parameter) Argumentele poziționale sunt argumente care trebuie incluse în poziția sau ordinea corespunzătoare. Primul argument pozițional trebuie întotdeauna să fie listat mai întâi atunci când funcția este apelată. Al doilea argument pozițional trebuie să fie enumerat al doilea, iar al treilea argument pozițional al treilea etc.
  • cheie (keyword parameter) De exemplu, pentru functia print(), sep sau end sunt chei de argument.

    Operatori aritmetici

    • - (scadere)

    • / (impartire cu zecimale), ex: 7/2 = 2.3333333333333335 (rezultatul este intotdeauna un float)

    Variabile



    Varabilele sunt ca niste recipiente care retin valori de diferite tipuri. De exemplu, o sticla poate contine lapte sau suc. Din aceasta perspectiva, sticla este numele recipientului, iar continutul (laptele, sucul) poate fi vazut ca valoarea recipentului (una e valoarea sau pretul laptelui, una este pretul sucului).

    Numele variabilelor in Python trebuie sa respecte niste reguli:
  • trebuie sa contina doar litere mici (a-z), litere mari (A-Z) si _ (underscore) - si nu doar litere/caractere latine (adica se poate folosi de exemplu: û)
  • trebuie sa inceapa cu o litera sau _ (underscore)
  • nu trebuie sa fie un nume rezervat (keyword): 'False', 'None', 'True', 'and', 'as', 'assert', 'break', 'class', 'continue', 'def', 'del', 'elif', 'else', 'except', 'finally', 'for', 'from', 'global', 'if', 'import', 'in', 'is', 'lambda', 'nonlocal', 'not', 'or', 'pass', 'raise', 'return', 'try', 'while', 'with', 'yield'


    • Python este un limbaj tipizat dinamic (dynamically-typed language) (nu e nevoie de declarearea unui variabile; unei variabile se asigneaza o valoare):
  • nume_variabila = valoare_variabila
  • nume_variabila = expresie
    • Observatie:
  • nu este permisa folosirea unei variable care nu exista, nu i-a fost asignata o valoare


    • Numele variabilelor in Python sunt case-sensitive (de exemplu, variabile rezultat si Rezultat sunt vazute ca 2 variabile diferite).
        rezultat=72
        Rezultat=31
        _=1
        u=2
        print(rezultat)
        print(Rezultat)
     Rezultatul va fi:
        72
        31
        1
        2


    Conventii de nume:
  • numele de variabile ar fi bine sa inceapa contina doar litere mici, iar cuvintele sa fie despartite pentru liziblitate prin _ (underscore): cal, cal_de_lupta
  • numele de functii ar trebuie sa respecte acelasi conventii ca numele de variabile
  • se poate folosi si conventia ca in loc de _ sa se puna fiecare cuvant al variabilei cu litera mare, in afara de primul cuvant: cal, calDeLupta


    • Se poate asigna si o expresie. De exemplu:
        text1 = "a_" + str(1)
        text2 = "a_" + "57"
        x = 2 + 7print(text1)
        print(text2)
        print(x)
     Rezultatul va fi:
        a_1
        a_57
        10

    Cateva observatii de caractere



    Calculatoarele stocheaza caracterele ca numere. Există mai multe moduri posibile de codificare a caracterelor. Exemple:
    • ASCII: (American Standard Code for Information Interchange) codifica alfabetul latin și unele dintre derivatele acestuia (are spartiu pentru 256 de caractere)
    • UNICODE: codifica toate alfabetele, folosind diferite moduri de codare:
      • UCS-4: (Universal Character Set): UCS-4 folosește 32 de biți (patru octeți) pentru a stoca fiecare caracter, iar codul este doar numărul unic al punctelor de cod Unicode.
        Un fișier care conține text codificat UCS-4 poate începe cu un BOM (byte order mark), o combinație de biți care nu poate fi imprimată care anunță natura conținutului fișierului.
      • UTF-8: (Unicode Transformation Format) foloseste mai putin memorie, pentru ca folosește doar cati biti are nevoie pentru a reprezenta un punct de cod (8 biti pentru caracterele latine, 16 biti pentru non-latine, 24 biti idiograme (China, Japonia, Korea ) )

      • A B A and B
      True True True
      True False False
      False True False
      False False False

          b=10
          c=a and b
          print(c)

      True True True
      True False True

      True False

    Capitol III



    Operatori pe biti

    Operatori pe biti:
    • & (si/conjunctie)

    Exemplu:
        Să calculăm 13 & 151.
        Reprezentarea lui 13 este 0000000000001101.
        Reprezentarea lui 151 este 0000000010010111
        0000000000001101 &
        0000000010010111
        Se obține: 0000000000000101, adică 5 Deci: 13 & 151 == 5

    • | (sau/disjunctie)


    Exemplu:
        Să calculăm 13 & 151.
        Reprezentarea lui 13 este 0000000000001101.
        Reprezentarea lui 151 este 0000000010010111
        0000000000001101 |
        0000000010010111
        Se obține:0000000010011111, adică 159 Deci: 13 | 151 = 159
    • ^ (or exclusiv)


    Exemplu:
        Să calculăm 13 ^ 151.
        Reprezentarea lui 13 este 0000000000001101.
        Reprezentarea lui 151 este 0000000010010111
        0000000000001101 ^
        0000000010010111
        Se obține: 0000000010011010, adică 154 Deci: 13 ^ 151 = 154
    • ~ (negatie)


    Exemplu:
        ~ 133 = -134
        Reprezentarea lui 133 este 0000000010000101. Prin complementare se obține 1111111101111010.
        Aceasta este reprezentarea în memorie a lui -134.
        Pentru a verifica, îl reprezentăm conform celor de mai sus pe -134:
        - reprezentarea lui 134 este 0000000010000110
        - prin complementare se obține 1111111101111001
        - adunăm 1 și obținem 1111111101111010
    • << (deplasare spre stânga / binary left shift)


    Exemplu:
        Să calculăm 13 << 3.
        Reprezentarea lui 13 este 0000000000001101. Deplasând toți biții spre stânga cu 3 poziții se obține:
        0000000001101000, adică 104.
        Să observăm că 104 este egal cu 13 * 2^3.
        În general n << k este n * 2^k.
        Pentru a calcula 2^n putem folosi operația 1 << n.

     • >> (deplasare spre dreapta / binary right shift)


    Exemplu:
        Să calculăm 133 >> 3.
        Reprezentarea lui 133 este 0000000010000101. Deplasând toți biții spre dreapta cu 3 poziții se obține:
        0000000000010000 adică 16.
        Să observăm că 16 este egal cu 133 / 23.
        În general n >> k este n / 2k
    Simplificat:
    • & necesită exact doi 1 pentru a furniza 1 ca rezultat; echivalent: A*B sau min(A,B)
    • | necesită cel puțin un 1 pentru a furniza 1 ca rezultat; echivalent cu: max(A,B)
    • ^ necesită exact un 1 pentru a furniza 1 ca rezultat



    Fie x un numar intreg. Atunci, avem:
    • x & 1 = x
    • x & 0 = 0
    • x | 1 = 1
    • x | 0 = x
    • x ^ 1 = ~x
    • x ^ 0 = x
    Presupunem ca avem o valoare. Si acea valoare, este formata din biti. Putem sa vedem ca fiecare bit este o stare. Sa zicem ca avem valoarea 5, in binar fiind reprezentat ca 101. Si acum, daca pe pozitia 3 din reprezentarea binara a numarului nostru este 1, atunci sistemul care verifica asta, va porni un cantecel in limba franceza.

    1. Verificare bit (cu &):

    Acum, sa invatam sistemul/programul sa faca verificarea de care am zis mai sus. In primul rand creem o masca cu valoare egala cu 2^(3-1), adica 4 (in binar 100):
        masca = 4 #vrem sa verificam bitul de pe pozitia 3
        numarul_nostru = 5 #101
        print(numarul_nostru & masca) # va returna o valoare masca
        alt_numar = 3 #011
        print(alt_numar & masca) # va returna 0

    2. Setare bit (cu |):

    Acum, sa invatam sistemul/programul sa seteze valoarea 1 pe bitul de pe pozitia 3. Folosind aceeasi masca, cu valoare egala cu 2^(3-1), adica 4 (in binar 100):
        masca = 4 #vrem sa verificam bitul de pe pozitia 3
        alt_numar = 3 #011
        print(alt_numar & masca) # va returna o valoare masca
        set_1 = alt_numar | masca
        print(set_1)

    3. Resetare bit (cu & si ~ pe masca):

    Acum, sa invatam sistemul/programul sa reseteze valoarea 1 pe bitul de pe pozitia 3. Folosind aceeasi masca, cu valoare egala cu 2^(3-1), adica 4 (in binar 100):
        masca = 4 #vrem sa verificam bitul de pe pozitia 3
        numarul_nostru = 5 #101
        print(numarul_nostru & ~masca) # 001, deci 1
    4. Negarea bit (cu ^ - adica, daca e 0 sa devina 1 si daca e 1 sa devina 0):

    Acum, sa invatam sistemul/programul sa reseteze valoarea 1 pe bitul de pe pozitia 3. Folosind aceeasi masca, cu valoare egala cu 2^(3-1), adica 4 (in binar 100):
        masca = 4 #vrem sa verificam bitul de pe pozitia 3
        numarul_nostru = 5 # 101
        print(alt_numar ^ masca) # 001, deci 1






    Functii

    Un spaţiu de nume, numit şi namespace sau scope, este folosit pentru izolarea codului. Un spaţiu de nume nu este creat în mod direct, ci prin intermediul altor structuri care determină crearea unui astfel de spaţiu. Structurile care determină crearea unui spaţiu de nume sunt funcţiile şi clasele. Structuri precum if, with, while, for etc nu determină crearea unui nou spaţiu de nume, deci variabilele definite în interiorul acestora vor fi vizibile şi după încheierea lor. Un spaţiu de nume ne determină componentele la care avem acces (variabile, funcţii, clase etc). Spaţiile de nume pot fi incluse unul în altul, iar spaţiul inclus va avea acces atât la componentele definite în interiorul său, cât şi la cele definite în spaţiile de nume superioare (care îl includ). Spaţiul de nume în care ne găsim odată cu deschiderea unui nou fişier Python poartă numele de spaţiu global, iar orice spaţiu de nume inclus în acesta va fi numit spaţiu de nume local. O variabilă definită în spaţiul de nume global se numeşte variabilă globală, iar o variabilă definită în spaţiul de nume local se numeşte variabilă locală. La ieşirea dintr-un spaţiu de nume local, toate componentele acestuia vor fi şterse din memorie. Dacă denumirea unei variabile locale coincide cu una dintr-un spaţiu de nume superior, ea va "masca" acea variabilă. Numele unei variabile este doar atât, un nume, referinţa (adresa în memorie) acesteia este cea care contează pentru Python. Acest lucru ne împiedică totuşi să mai accesăm acea variabilă globală din spaţiul de nume local, deci trebuie să aveţi grijă ce mascaţi. Funcţii În lecţiile precedente am tot utilizat funcţii pentru a efectua diverse operaţii, astăzi vom învăţa cum să ne creăm propriile funcţii. O funcţie ne permite gruparea mai multor linii de cod cu scopul de a reduce duplicarea codului.
    Putem defini o functie astfel:
        def nume_functie(arg1, arg2):
        """ informatii despre functie """
         print(arg1, arg2)
         return 0
    Unde def este cuvântul cheie care indică definirea unei noi funcţii, nume_functie este numele funcţiei ales de noi (cu aceleaşi restricţii şi recomandări ca denumirea unei variabile), iar între paranteze vor fi precizaţi parametrii funcţiei (dacă nu există, lăsăm parantezele goale). Comentariul delimitat de ghilimele triple (""") şi precizat imediat sub definiţia funcţiei poartă numele de docstring şi este folosit pentru a oferi detalii despre funcţia în care se află (detaliile includ informaţii despre rolul funcţiei, tipurile de date şi descrierea argumentelor şi tipul de date returnat). Prezenţa unui docstring nu este obligatorie, însă aceasta poate face codul mult mai clar şi poate uşura menţinerea acestuia. Mai mult, prezenţa unui docstring pentru fiecare funcţie este recomandată şi de standardul PEP8. Sintagma return este de asemenea opţională. Aceasta este folosită pentru a returna o valoare la finalul execuţiei funcţiei. Dacă această instrucţiune nu este precizată, funcţia nu va returna nimic, ceea ce este echivalent cu a spune return None, unde None este un cuvânt cheie în Python care indică absenţa unei valori (diferit de 0 care este un număr întreg). De asemenea, pot exista mai multe instrucţiuni return într-o funcţie, însă prima întâlnită va fi cea executată. Odată executată instrucţiunea return, funcţia se va termina chiar dacă mai există cod sub aceasta. Observaţi caracterul : de la finalul definiţiei funcţiei, care ne indică începerea unui nou bloc de cod şi prezenţa indentării pentru blocul de cod atribuit funcţiei. Acestea fac parte din sintaxă şi sunt obligatorii. Parametrii unei funcţii sunt variabile locale în aceasta, iar orice variabilă definită în interiorul unei funcţii va fi, de asemenea, locală. Asta înseamnă că, la terminarea funcţiei, variabilele respective nu vor mai fi vizibile în spaţiul de nume exterior. Pentru a apela o funcţie este suficient să îi precizăm numele urmat de paranteze rotunde (în interiorul cărora pot fi specificate valori pentru parametrii acesteia, dacă este cazul).
        >>> nume_functie("a", 3)
        a 3
    Dacă vrem să păstrăm valoarea returnată de o funcţie, este suficient să o atribuim unei variabile
        >>> x = nume_functie("a", 3)
        a 3
        >>> x
        0
    Parametrii de mai sus poartă numele de parametrii poziţionali, iar prezenţa lor ne obligă să le specificăm o valoare în momentul apelării funcţiei. Mai mult, ordinea în care dăm valorile contează (de aici numele de poziţionali). Un alt mod de a apela funcţia este specificând numele fiecărui parametru şi valoarea atribuită acestuia. Parametrii specificaţi astfel la apelare poartă numele de keyword arguments, şi ne permit să specificăm parametrii în orice ordine dorim (chiar dacă sunt parametrii poziţionali):
        >>> nume_functie(arg2=3, arg1="a")
        a 3






    Module

    Asemenea funcţiilor, modulele reprezintă o modalitate de a organiza codul cu scopul de a uşura reutilizarea acestuia. Spre deosebire de funcţii, însă, unde erau grupate doar blocuri de cod, modulele pot conţine funcţii, clase şi variabile. Mai mult, modulele pot importa şi utiliza componentele altor module. Altfel spus, un modul este un fişier Python care poate fi importat în alte module cu scopul de a reutiliza structuri de cod definite în acesta. Pentru a crea un modul trebuie doar să salvăm fişierul care conţine codul de interes pentru noi într-un fişier cu extensia .py. Acum dacă deschidem un alt fişier (alt modul) şi vrem să refolosim cod din primul modul, suntem obligaţi să îl importăm mai întâi. Importul unui modul se face astfel:
         import nume_modul
    unde nume_modul reprezintă numele modulului pe care vrem să îl importăm. Şi putem apela o funcţie din interiorul acestuia prin:
        nume_modul.nume_functie()
